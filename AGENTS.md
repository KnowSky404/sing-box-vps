# sing-box-vps 项目开发规约

本项目旨在开发一个针对 VPS 环境的 `sing-box-vps` 一键安装与管理脚本。所有后续开发活动必须严格遵守本文件中的约定。

## 1. 项目背景
`sing-box-vps` 是一个多功能的通用代理平台。由于其配置相对复杂，普通用户在 VPS 上部署时门槛较高。本项目通过 Shell 脚本实现自动化安装、配置生成及服务管理，降低使用门槛。

## 2. 核心原则
- **极简主义**：不引入不必要的依赖，优先使用系统自带工具（如 `curl`, `sed`, `grep`, `systemctl`）。
- **健壮性**：严格的错误检查，确保在任何步骤失败时都能安全退出并给出明确提示。
- **兼容性**：优先支持主流 Linux 发行版（Debian 11+, Ubuntu 20.04+, CentOS 7+/Stream, AlmaLinux, Rocky Linux）。
- **安全性**：遵循最小权限原则，保护用户隐私和密钥信息，不留后门。

## 2.1 文档检索约束

- 只有在 Context7 无法提供结果、结果明显不完整，或无法覆盖当前问题时，才允许退回使用 `web search`、`web fetch` 或其他网页抓取方式。

## 3. Shell 开发规范
### 3.1 基础约定
- **解释器**：统一使用 `#!/usr/bin/env bash`。
- **错误处理**：脚本开头必须包含 `set -euo pipefail`，确保管道错误也能被捕获。
- **编码**：必须使用 UTF-8 编码。
- **缩进**：统一使用 2 个空格，禁止使用 Tab。

### 3.2 变量与函数
- **变量命名**：全局变量使用大写字母加下划线（如 `CONFIG_PATH`），局部变量使用小写字母加下划线（如 `local temp_file`）。
- **局部变量**：在函数内部定义变量时，必须使用 `local` 关键字。
- **函数命名**：使用小写字母加下划线，且命名应具有描述性（如 `install_dependencies`）。
- **常量定义**：脚本顶层定义所有的常量、颜色代码和文件路径。

### 3.3 输出与交互
- **日志输出**：区分 `INFO`, `WARN`, `ERROR`, `SUCCESS` 状态，使用不同的颜色标识。
- **交互提示**：所有的用户输入必须经过有效性校验，并提供合理的默认值。
- **静默模式**：考虑支持环境变量或参数触发非交互式安装（CI/CD 或自动化部署需求）。

## 4. 架构与流程约束
- **依赖检查**：在执行任何操作前，必须检查当前用户权限（root）以及必要工具是否安装。
- **幂等性**：脚本应支持重复执行而不破坏系统状态（例如：如果服务已安装，应提示更新或重新配置）。
- **配置管理**：`sing-box` 的配置文件应采用模块化思路生成，方便后期维护 JSON 结构。
- **备份机制**：修改关键系统文件或已有配置前，必须进行 `.bak` 备份。
- **开发验证工作流**：涉及 `install.sh`、`uninstall.sh`、`configs/`、`utils/` 或远程验证框架本身的改动时，默认先运行 `bash dev/verification/run.sh`，命中远程规则时必须执行真实远程验证，不得只停留在本地静态检查。
- **远程验证目标配置**：远程验证默认优先读取 `dev/verification-target.env`；若该文件存在，后续开发与修复应直接复用，不要在新对话里重新设计另一套入口。兼容的回退方式才是 `VERIFY_REMOTE_HOST_ALIAS` 或 `VERIFY_REMOTE_HOST`/`VERIFY_REMOTE_USER`。
- **本地测试机别名约定**：本仓库当前约定的 SSH 测试机别名为 `sing-box-test`；若本地 `dev/verification-target.env` 已配置该别名，则应直接使用该目标机执行远程验证。
- **版本规范**：`install.sh` 中的 `SCRIPT_VERSION` 必须使用 `YYYYMMDDXX` 格式，其中前 8 位为日期，后 2 位为当日迭代序号；同步展示给用户的脚本版本信息时也必须保持一致。
- **版本递增时机**：当一次完整的对话轮次结束且该轮次实际产生了代码或脚本文件修改时，必须同步递增一次脚本版本号；同一轮对话内即使拆分为多个 commit，也只能递增一次版本号，不按 commit 数量递增；若该轮对话未产生代码修改，则不更新版本号。

## 5. 文件组织结构
- `install.sh`: 用户分发的一体化安装与管理脚本主入口，也是运行时单一真源。
- `docs/`: 存放设计文档、实现计划等研发过程资料。
- `configs/`: 存放配置模板。
- `utils/`: 存放通用工具函数（如颜色输出、系统检查）。
