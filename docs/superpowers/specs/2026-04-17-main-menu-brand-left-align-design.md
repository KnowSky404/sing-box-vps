# 主菜单品牌区左对齐与头部信息回收设计

- 日期：2026-04-17
- 适用范围：`install.sh` 主菜单头部与主菜单页脚
- 目标：修正当前主菜单“品牌区居中但菜单左对齐”的视觉割裂问题，将作者与项目地址并回顶部品牌区，同时保持头部尽量紧凑

## 1. 背景

上一轮终端 UI 压缩将作者与项目地址下放到了主菜单底部，虽然减少了头部高度，但也带来了新的版式问题：

- 顶部 Logo/标题区域仍然是居中展示
- 菜单分区和菜单项主体保持左对齐
- 顶部品牌信息与底部作者/项目信息被拆成两块，整体识别感被削弱
- 在宽终端中，主菜单中间和右侧留白较多，视觉重心偏散

本轮不重做整套终端 UI，只处理主菜单品牌区的对齐方式和信息归位。

## 2. 设计目标

- 主菜单顶部品牌区改为左对齐，和菜单主体共享同一视觉起点
- 作者与项目地址重新并回顶部品牌区，不再在底部单独显示
- 保留当前菜单分区、选项顺序、业务行为和子菜单紧凑风格
- 头部信息回收后仍保持紧凑，不回退到早期过高的头部样式

## 3. 非目标

- 不调整主菜单分区结构和菜单编号
- 不改动任何菜单动作映射或业务逻辑
- 不重新设计子菜单头部样式
- 不引入新的配色体系、边框样式或运行时依赖

## 4. 推荐方案

采用“主菜单专用左对齐品牌块”的方案：

- 主菜单不再使用当前通用的居中 `render_page_header()` 作为顶部头部
- 为主菜单单独输出一个左对齐品牌块
- 品牌块内集中展示标题、作者、项目地址、描述和版本
- 原有底部作者/项目地址页脚移除，避免信息分裂和重复

该方案可以直接解决对齐割裂问题，同时比恢复旧版多段头部更克制。

## 5. 页面结构

### 5.1 主菜单顶部品牌块

主菜单顶部调整为一个紧凑的左对齐品牌块，信息顺序固定为：

1. `sing-box-vps 一键安装管理脚本`
2. `作者: KnowSky404 · 项目: https://github.com/KnowSky404/sing-box-vps`
3. `专为 VPS 稳定部署与安全运维设计 · 版本: <SCRIPT_VERSION>`

该品牌块与后续菜单分区共享同一左边界，不再单独追求居中展示。

### 5.2 菜单主体

主菜单分区与菜单项保持当前结构不变：

- 部署管理
- 服务控制
- 连接与诊断
- 脚本维护

仅调整其与顶部品牌块的衔接关系，不改动顺序、文案和编号。

### 5.3 主菜单底部

移除主菜单底部现有的：

- `作者: KnowSky404`
- `项目: https://github.com/KnowSky404/sing-box-vps`

底部不再承担品牌信息展示职责，避免形成“顶部标题、底部署名”的割裂结构。

## 6. 实现方式

- `show_banner()` 改为渲染主菜单专用品牌块，输出采用左对齐
- `render_page_header()` 保持现状，继续供子菜单使用
- `render_main_menu_footer()` 移除作者和项目地址输出，必要时保留空实现或由调用方不再依赖该函数
- 主菜单现有的分区标题、菜单项渲染 helper 继续复用，不另起一套菜单布局逻辑

这样可以把变更范围严格限制在主菜单头部/页脚层，不影响其他菜单页面。

## 7. 信息展示策略

### 7.1 顶部保留

主菜单顶部统一展示：

- 标题
- 作者
- 项目地址
- 描述
- 版本

### 7.2 底部移除

主菜单底部不再展示：

- 作者
- 项目地址

### 7.3 子菜单保持不变

子菜单继续沿用当前紧凑头部规则：

- 保留标题
- 在需要时保留副标题
- 不恢复作者
- 不恢复项目地址
- 不改成主菜单这种左对齐品牌块

## 8. 验证范围

实现完成后至少验证：

- 主菜单顶部出现左对齐品牌块
- 顶部品牌块包含标题、作者、项目地址、描述和版本
- 主菜单底部不再重复作者和项目地址
- 主菜单分区和选项顺序保持不变
- 子菜单仍保持当前紧凑头部，不出现作者和项目地址回流
- 主菜单相关渲染测试通过，并更新对应断言

## 9. 风险与控制

- 风险：品牌信息回收后头部重新变高
  - 控制：采用单块紧凑输出，限制在约 3 行有效信息内，不新增额外装饰行
- 风险：只调整主菜单可能和子菜单形成新的风格差异
  - 控制：明确将主菜单视为唯一品牌入口，子菜单继续承担操作页角色
- 风险：原有测试仍然假设作者/项目地址出现在底部
  - 控制：同步调整主菜单渲染测试，改为验证顶部出现、底部不再出现

## 10. 结论

本轮应以“品牌信息重新收拢到主菜单顶部，并统一改为左对齐”为核心，优先修正当前视觉割裂和留白别扭的问题。实现上只针对主菜单做专用品牌块，不回退到旧版高头部，也不打乱现有菜单结构与子菜单紧凑策略。
